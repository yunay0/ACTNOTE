# Features

CAP-001	파일 업로드	  • 회의 녹음 파일(mp3, wav, mov 등)을 업로드하여 자동으로 STT 분석 파이프라인을 시작.

  • 최대 500MB 용량과 120분 분량까지 지원하며 업로드 중 실시간 프로그레스 바를 제공함.

  • 파일 형식이나 용량 초과 등 검증 실패 시 에러 사유를 포함한 팝업을 노출하여 재시도를 유도함.

DRAFT-001	AI 회의록 수정 	  • 관리자는 AI가 생성한 초안 내용을 직접 수정·삭제·병합하여 실제 회의 맥락에 맞게 최종 결과물을 보완함

  • 요약문 내 액션 아이템의 담당자 변경 ·마감 기한 수정 · 태깅된 관련 문서 변경 가능

  • AI 분석의 불확실성을 사용자가 직접 수정하여 → 최종 발행될 회의록의 신뢰도와 정확성을 확보함

DRAFT-002	AI 요약 생성	  • STT 전사 완료 후 GPT-4o 모델이 회의 내용을 자동 분석하여 요약, 결정사항, 액션 아이템 등을 구조된 요약문으로 자동 생성  → 회의 흐름을 한눈에 파악할 수 있다.

  • 회의 유형 템플릿에 따라 알맞는 요약을 제공합니다. (예: 스프린트/ 1:1 / 클라이언트 미팅 등)

  • 회의의 핵심 흐름을 파악할 수 있도록 전체 대화 내용을 압축 정리하며, 후속 분석 기능들의 기초 데이터 역할을 수행합니다. 

DRAFT-003	결정사항 추출	  • AI가 회의 전사에서 결정 패턴을 자동 감지하여 별도 섹션으로 정리함. (예:  "~하기로 했다", "~로 결정", "let's go with...", "we agreed on..." 등)

  • 회의 중 합의된 핵심 결정사항(Decisions)을 추출.

  • 추출된 결정사항은 책임자가 추가·수정·삭제 가능.

DRAFT-004	액션 아이템 추출	  • AI가 회의 전사에서 패턴을 감지하여 대화 맥락을 기반으로 액션 아이템을 자동 추출 및 구조화(task, assignee, due_date)합니다. (예:  "~까지 해주세요", "I'll take care of...", "action item" 등)

  • 책임자가 Draft(초안) 화면에서 테이블 형태로 모든 필드를 수정 가능하며, 이 구조가 그대로 Notion DB 티켓 생성에 사용됨

DRAFT-005	담당자 태그	  • 추출된 액션 아이템에 담당자를 태그하여 책임을 명확히함.

  • 액션 아이템에 태그된 담당자를 Notion DB 티켓 담당자 필드와 자동 맵핑함.

  • 책임자가 워크스페이스에 등록된 멤버 목록에서 드롭다운으로 선택·수정 가능함

DRAFT-010	식별되지 않은 사용자 추측하여 선택(태그)	  • STT 결과에서 화자 분리 후 워크스페이스 멤버와 매칭되지 않는 화자가 있을 경우 AI가 발언 패턴 기반으로 멤버 후보를 추측해 제안하고, 사용자가 이를 확인하고 선택·확정함

  • → 핵심기능의 예외처리이기때문에 핵심기능으로 포함

STATUS-001	회의록 목록 (발행 상태 관리)	  • 회의록을 Draft(초안), Ready(검토완료), Published(발행완료) 3가지 상태로 관리합니다.  

  • 카드에서 제목, 날짜, 참여자(최대 3명), 액션 아이템 개수를 즉시 확인합니다.  

  • 상태·날짜·프로젝트별 필터와 10개 단위 페이지네이션을 제공합니다.

PUB-001	회의록 승인 및 발행	  • 책임자가 Draft를 최종 검토(필수) 후 → "Publish" 버튼 클릭 시 필수 필드(제목, 요약, 액션 아이템 최소 1개) 검증 후 발행.

  • 발행 시 상태가 'published'로 변경되며, 노션 API를 통해 지정된 페이지/DB에 회의록과 티켓이 자동 생성/업데이트됨 .

STATUS-002	회의록 삭제 (발행 취소)	  • 책임자는 AI 분석 알림을 받고, 분석 결과를 확인 후 → 회의록 Draft(초안)을 삭제할 수 있다.

  • 책임자는 발행된 회의록을 삭제 (발행 취소)하는것도 가능함. 

  • 삭제 시 확인 팝업 노출. 

SEC-001	모델 학습 옵트아웃	  • 프라이버시 보호 강화를 위한 기능으로, 

  • 사용자 데이터를 AI 모델 학습에 사용하지 않도록 기본값 ON 설정 → 사용자 데이터 미사용(학습 안함) 선택

  •[워크스페이스 설정]에서 ON/OFF 토글 가능하며, opt_out 플래그를 DB에 저장하고 STT·LLM API 호출 시 training_data=false 파라미터 자동 전달.

PLATFORM-002	웹 앱	  • ACTNOTE 핵심 사용자 인터페이스로, 모든 플로우의 실행 환경이 웹 앱에서 이루어진다.

  • 파일 업로드, Draft 검토·수정, 발행, 회의록 목록 조회, 설정(Notion 연동, 워크 스페이스 초대 등) 전체 워크플로우를 데스크톱 브라우저 환경에서 제공.

SEC-002	회원가입/로그인	  • (회사) Google 계정을 통해 소셜 회원가입/로그인 

  • 서비스 이용정책 및 개인정보 처리방침 필수 동의 후 이용가능

SEC-010	설정	  • 사용자 기본설정(이름, 프로필 이미지, 계정) 및 워크스페이스 설정(팀원 초대, 프로젝트, 통합 서비스) 관리 

WS-001	워크스페이스 생성	  • 첫 로그인(SEC-002) 완료 직후 Admin에게 워크스페이스 생성 플로우 자동 진입.

  • 워크스페이스명, 팀 규모(선택), 사용 언어(기본값: 영어) 입력 후 생성.

  • 생성 완료 시 Admin 권한으로 해당 워크스페이스에 자동 귀속되며, 초대 링크가 생성됨.

  • 이미 워크스페이스가 있는 경우(재로그인 등) 이 단계는 스킵, 기존 워크스페이스 목록으로 이동.

WS-002	워크스페이스 참여	• 초대 링크 클릭 시 구글 계정 로그인 후 워크스페이스 진입

• 해당 워크스페이스의 Notion 연동 상태를 자동 확인(INTEG-003 연동 여부). 미연동 시 역할에 따라 안내 분기(INTEG-005).

WS-003	참여자 역할 권한 부여	• Admin이 워크스페이스 멤버에게 Admin / Member 2가지 역할 중 하나를 지정·변경.

• 역할별 권한 범위: Admin — 워크스페이스 설정·연동 설정·권한 부여 전체+회의 생성·발행·수정 / Member — 회의록 생성 및 회의 결과 읽기 전용.

• 권한 변경은 워크스페이스 설정 내 멤버 관리 화면에서 수행. 변경 즉시 반영.

WS-004	워크스페이스 멤버 관리 (초대/삭제)	  • Admin이 워크스페이스 멤버를 초대하거나 삭제하는 기능. 

  • WS-003(역할 권한 부여)과 연결되며, 멤버 삭제 시 해당 멤버의 접근 즉시 차단. 

  • 워크스페이스 초대는 이메일 발송/초대링크 생성 및 복사(ws-005)로 연결됨.

WS-005	워크스페이스 초대 (링크 생성 / 메일 발송)-여기서부터	• Admin이 워크스페이스 초대용 고유 링크를 생성. 생성된 링크는 복사하거나 Gmail로 발송하는 두 가지 방식으로 전달 가능.

• 링크 전달 방식 

      ◦ ① 링크 복사: 클립보드에 복사 후 직접 공유. 

      ◦ ② 이메일 발송: 초대할 멤버의 Gmail 주소 입력 시 초대 메일 자동 발송(gmail) — 메일 본문에 초대 링크 포함.

• 초대 링크 클릭 흐름: ACTNOTE 진입 → Google 계정으로 소셜 로그인(SEC-002) → 워크스페이스 참여(WS-002) 진입.

• 링크는 1회용 또는 만료 기간 설정 가능 여부 추후 개발팀과 협의 필요.

• WS-004(멤버 초대/삭제)의 "초대" 액션과 연결 — WS-004에서 초대 시작 → WS-005에서 링크 생성·발송 처리.

MTG-001	회의록 생성	• 홈에서 "새 회의록 생성" 버튼 클릭 시 새 회의록 생성 및 기본 정보(메타정보 입력(MTG-002)) 플로우 시작.

MTG-002	회의 메타정보 입력	 • 회의 메타정보 입력 ( 필수: 회의명, 일시, 작성자, 참석자, 책임자, 회의유형 / 선택: 이외 상세 정보)

  • 입력된 정보는 회의록 발행 후 Notion DB push 시 해당 필드에 자동 매핑(INTEG-003).

MTG-003	회의록 분석	  • 파일 업로드(CAP-001) 완료 후 "분석하기" 버튼 활성화, 클릭 시 STT → AI 초안 생성(DRAFT-002~006) 파이프라인 실행.

  • 실행 전 노션 연동 상태를 체크하여 미연동 시 안내 권유를 띄우며, 분석은 백그라운드에서 진행됨

  • 분석 완료 시 NOTI-001(분석 완료 알림) 자동 발송 및 Draft 상태로 결과 화면 진입

MTG-004	회의록 템플릿 프롬프트 설계	  • 회의 유형(스프린트, 기획, 회고 등)에 따라 AI 회의록의 요약 구조, 섹션 구성, 액션 아이템 추출 방식 등을 사전에 정의하여 일관된 형태의 회의록 생성을 가능하게 하는 기능

NOTI-001	분석 완료 알림	  • AI 분석 파이프라인이 완료되거나 실패했을 때 작성자, 책임자, 식별되지 않은 사용자 중 앞에 속하지 않고, 액션아이템 할당된 사람에게 실시간 인앱 알림, 메일을 발송합니다.   

  • 성공 시에는 검토를 유도하는 메시지를, 실패 시에는 재업로드 안내 메시지를 제공합니다.  

  • 사용자가 알림을 클릭하면 즉시 해당 회의록의 Draft 결과 화면으로 연결되어 빠른 검토를 도움.

UX-001	확인 팝업	  • 사용자가 메타정보 입력 시, 도중에 화면을 이탈하려 할 때 확인 팝업 노출.
